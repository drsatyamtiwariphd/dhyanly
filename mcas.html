How It Works
The MCAS pipeline — 8 stages, all on your device
01
App Detection
MCAS detects when a monitored social or entertainment app moves to the foreground. Monitoring begins immediately.
UsageStatsManager · AccessibilityService
02
Screen Capture
A frame is captured every 20 seconds. Frames are held briefly in memory for analysis, then immediately deleted — never saved to disk.
MediaProjection API · JPEG compression
03
On-Device OCR
Text visible on screen — captions, hashtags, comments, overlays — is extracted and tokenised for keyword classification.
Google ML Kit Text Recognition v2
04
Image Classification
Each frame passes through a lightweight neural network detecting faces, emotional scenes, violence, fast-cut patterns, and lifestyle imagery.
MobileNetV3 · TensorFlow Lite INT8
05
Content Classification
OCR tokens and image scores are combined into a category distribution per frame: Addictive, Emotional, Educational, Entertainment, Positive.
Weighted fusion · Rule-based engine
06
Behaviour Analysis
Patterns across frames are tracked — repetition loops, emotional drift, addiction spirals, night vulnerability, and duration thresholds.
Temporal pattern engine · Scoring model
07
Yogic Intervention
When a threshold is crossed, MCAS triggers a personalised intervention — Nadi Shodhana, grounding, a breath pause, or a digital sabbath suggestion.
Svara Yoga · Pranayama library
08
Encrypted Storage + Report
Session summaries are stored encrypted on your device. Daily reports surface awareness insights — framed as data, not shame.
AES-256 · Room Database · Local only
